# Security

# Intended users of this repository
This repository is intended for:
- Me (Connor Lafferty) to submit coursework.
- The professor (Matthew Lamoureux) to view, run, and grade the code and assignments.

# Risk assessment

**Academic integrity issues**
  If this repo was used by a classmate all the answers to the homework assignments could be located here once I finish the assignments.

At the moment, the main concerns are if I accidentally push secrets or private data like my passwords, or if another student gained access and cheated on all the homework assignments.

# Steps taken to secure the repository

**No secrets committed!**
  I avoid storing API keys/passwords in the repo and make sure I'm not pushing any to it accidentally.

**Repo access & workflow protections**
  - No `CODEOWNERS` file is made, since people would not need to pull the code to cheat on the homework. They could just view the lines that I've created and copy and paste them.


Since I trust the other students in the class to maintain a high academic standard, I have not privatized my repo. Also since I need to submit the assignments, I keep the repo public so my professor can view them. If this trust is too much to have in other students (my belief that they will not cheat off me), and the professor wishes for higher security, I will alter my repo's viewability to fit the regulations.
